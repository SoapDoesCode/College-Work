Web Content Accessibility Guidelines (WCAG)

What are they?

The Web Content Accessibility Guidelines are standards set out by the World Wide Web Consortium with the aim of making the web accessible to everyone (especially those with disabilities)

What do they cover?

The Web Content Accessibility Guidelines cover issues related to:

  • Vision
  • Hearing
  • Cognitive limitations
  • Learning disabilities
  • Limited movement
  • photosensitivity

What are the 4 principles of these guidelines?

The 4 principles of the Web Content Accessibility Guidelines are to ensure that web content is:

  • Perceivable
  • Operable
  • Understandable
  • Robust

What do these principles mean?

Perceivable

This principle states that information must be available to the user's senses (e.g., contain alt text, sufficient contrast between content and backgrounds, and being adaptable to accessibilitty settings).

Operable

Operability refers to the principle that user interface components and navigation must be usable by all users, regardless of device, operating system, or browser. This principle ensures users can interact with the interface without requiring actions that may not be possible for them to perform. This includes making all functionality and navigation accessible via keyboard.

Understandable

The principle of content being understandable means that information and the operation of the user interface must be clear and easy to comprehend for all users. This includes using text that is readable, navigation being predictable, and providing assistance that helps users avoid and correct any possible mistakes.

Robust

Robustness refers to the idea that web content must be compatible with current (and future) user agents, including assistive technologies, ensuring it can be reliably interpreted by a variety of tools. This involves following web standards and maintaining clean code to avoid errors that may render a site or page to be inaccessible.
